# templates

노트 작성을 시작할 때 **참고하는 프롬프트** 모음. 반드시 따라야 하는 틀이 아니다.
새 노트에 복사해 쓰되, **안 맞는 섹션은 지우고** 필요한 건 더한다.

## 유형

- [개념 노트](./concept.md) — 원리·개념 정리 (왜/어떻게 이해)
- [문제 풀이](./problem-solving.md) — 알고리즘·코딩 테스트 풀이
- [트러블슈팅](./troubleshooting.md) — 버그·삽질·해결 기록
- [치트시트](./cheatsheet.md) — 문법·명령어 빠른 참조

## 원칙

- **틀(mold)이 아니라 프롬프트(prompt)**: 채워야 하는 칸이 아니라, 답하면 좋은 질문. 안 맞으면 섹션을 지운다.
- 새로운 유형은 **실제로 그 유형의 노트가 생길 때** 여기에 추가한다.

> 트러블슈팅·치트시트는 아직 실제 노트 예시 없이 먼저 만든 것. 각 유형의 첫 노트를 써보며 모양을 다듬을 것.
