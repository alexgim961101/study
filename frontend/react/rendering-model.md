# 렌더링 모델 (Rendering Model)

컴포넌트는 함수, 렌더링은 그 함수의 재실행. React 원리의 뿌리.
실습: [counter-app](../../projects/counter-app)

## 코드 근거

- `App.tsx`: `const [count, setCount] = useState(0)`, `setCount(count + value)`
- 컴포넌트는 함수다: `function App() { ... }`, `const Viewer = (...) => { ... }`

## 정리

### 1. 컴포넌트 = 함수, 리렌더링 = 함수의 재실행

React가 화면을 "다시 그린다(re-render)"는 건, 그 컴포넌트 **함수를 처음부터 끝까지 다시 호출**한다는 뜻이다.
`+1` 클릭 → App 리렌더 → `App()` 재실행 → 이번엔 `count`가 `1`인 채로 `<Viewer count={1}/>`를 반환 → 화면 `1`.

### 2. 리렌더를 일으키는 건 setter 호출뿐

React는 변수를 **감시하지 않는다.** 그냥 `let count = 0`을 두고 `count++` 해봤자 아무도 "다시 그려"라고 알리지 않으므로 화면은 그대로다.
리렌더를 예약하는 유일한 방법은 **`setCount(...)` 호출**이다. 즉 `setCount`는 값을 그 자리에서 바꾸는 게 아니라, **"바뀌었으니 다시 그려"라고 React에 알리는(리렌더 예약) 통로**다.

> `useState` = ① 값 보관 + ② setter로 변경 알림. `let 변수`엔 ②가 없어서 화면이 안 바뀐다.

### 3. state 값은 함수 바깥(React 내부)에 저장된다

지역변수(`let count`)는 함수 실행이 끝나면 사라진다. 하지만 state는 **React가 함수 바깥에 보관**하므로, `App()`이 몇 번 재실행돼도 값이 유지된다.

- `useState(0)`의 초기값 `0`은 **마운트(첫 렌더) 때 딱 한 번**만 쓰인다. 이후 모든 리렌더에선 무시되고, React가 보관 중인 **현재 값**을 돌려준다.
- 구현 감각: 저장된 값이 있으면 그걸, 없으면(첫 렌더) 초기값을 쓴다. 어느 저장값이 어느 `useState`인지는 **호출 순서**로 매칭한다 → Hook을 `if`·반복문 안에서 부르면 안 되는(Rules of Hooks) 이유.

### 4. 부모가 리렌더되면 자식은 전부 리렌더된다 (기본 동작)

App이 재실행되면 그 안의 `<Viewer>`·`<Controller>`도 **모두** 재실행된다. Controller처럼 받은 값이 안 바뀐 것처럼 보여도 마찬가지다. React는 "props 안 바뀐 자식 건너뛰기"를 기본으로 하지 않는다.

- 건너뛰게 하려면 `React.memo`·`useCallback`으로 opt-in 한다. **기본은 전부 재실행.**
- ⚠️ **함수 재실행(리렌더) ≠ 실제 DOM 갱신.** React는 재실행 결과를 이전과 비교해 **진짜 바뀐 DOM만** 최소로 고친다. (→ [declarative-ui](./declarative-ui.md))

### 5. (적용) 어떤 값을 state로 둘까

**기준: 값이 (1) 시간에 따라 변하고 + (2) 그 변화가 화면에 반영돼야 하면 → state. 둘 중 하나라도 아니면 → 그냥 변수/상수.**

- 현재 카운트 값 → 변함 + 화면 반영 → **state**
- `+10` 버튼의 `10` → 절대 안 변함 → **상수/변수** (state로 둘 필요 없음)
- 변하긴 하지만 화면엔 안 쓰는 값(예: 타이머 ID)은 state가 아니라 `useRef`로 둔다. (나중 거리)

## 한 줄 요약

state가 바뀌면(= setter 호출) 컴포넌트 함수가 다시 실행되고, 그 결과로 화면이 갱신된다. state는 함수 **바깥**에 저장돼 재실행 사이에도 값이 유지된다.

## 실험

세 컴포넌트(App/Viewer/Controller) 본문에 `console.log`를 찍고 버튼을 눌러, 누가 몇 번 다시 실행되는지 눈으로 확인해보기. 부모 App뿐 아니라 자식들도 함께 다시 실행되는 것을 관찰.
