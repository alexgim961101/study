# State 스냅샷 (State as a Snapshot)

state는 "지금 이 렌더링 시점의 값". 가장 깊고 헷갈리는 주제 — 카운터가 최적의 실험대.
실습: [counter-app](../../projects/counter-app)

## 코드 근거

- `App.tsx`: `const onClickButton = (value) => { setCount(count + value) }`
- 여기서 `count`는 "이 핸들러가 만들어진 렌더링 시점"의 값이다.

## 정리

### 1. 스냅샷이란

하나의 렌더 동안 `count`는 **그 순간 값으로 고정된 사진 한 장**이다. 렌더가 실행되는 내내 절대 안 변한다(사실상 상수). `setCount`를 아무리 불러도 **이 렌더 안에서** `count`는 그대로고, 새 값은 **다음 렌더**에서만 보인다.

> 이유(렌더링 모델 연결): `setCount`는 값을 그 자리에서 바꾸는 게 아니라 **리렌더를 예약**할 뿐이다. 값은 다음 재실행 때 React가 새로 꺼내 준다.

**예약된 리렌더는 언제 실행되나?** `setCount`는 "리렌더 필요"를 큐에 등록만 하고, **지금 실행 중인 이벤트 핸들러가 다 끝난 뒤** React가 큐를 모아 한 번에 처리한다(= **배칭 batching**). 그래서 한 핸들러에서 `setCount`를 여러 번 불러도 리렌더는 **1번**이고, 핸들러가 도는 동안엔 `count`가 바뀔 수 없다 — 이것이 스냅샷인 이유다.

```
클릭 → 핸들러 실행(setCount는 큐에 등록만) → 핸들러 끝 → React가 큐 처리(App 재실행 → DOM 커밋) → 페인트
```

(참고: React 18+는 `setTimeout`·`fetch().then()` 안의 setState도 모아서 배칭한다 — automatic batching.)

### 2. 값 형태로 두 번 부르면? (count가 0일 때)

```tsx
setCount(count + 1)   // count는 0 → setCount(1)
setCount(count + 1)   // count는 여전히 0 → setCount(1)
```

`count`가 이 렌더 내내 0으로 고정이라 둘 다 `setCount(1)`. 결과는 `2`가 아니라 **`1`**.

### 3. 함수형 업데이트 — 진짜로 +2 하기

값 대신 **함수(업데이터 함수)**를 넘긴다. React가 처리 시점에 **최신 값**을 인자로 넣어 대신 호출한다(= 콜백).

```tsx
setCount(c => c + 1)   // React 실행: c=0 → 1
setCount(c => c + 1)   // React 실행: c=1 → 2   (앞의 결과가 들어옴)
```

→ 누적돼서 **`2`**.

### 4. 값 형태 vs 함수 형태

| | `setCount(count + 1)` | `setCount(c => c + 1)` |
|---|---|---|
| 넘기는 것 | 값 (고정된 결과 숫자) | 함수(레시피) "어떻게 바꿀지" |
| `count` / `c` | 이 렌더의 **스냅샷** | React가 넣는 **최신값** |
| 두 번 호출 | → `1` | → `2` |

한 줄: 값 형태 = "`1`로 해줘", 함수 형태 = "지금 값이 뭐든 +1 해줘".

### 5. (적용) 언제 함수 형태를 쓰나

**새 값이 "이전 state 값"에 의존하면 → 함수 형태(`c => ...`). 이전 값과 무관하면 → 값 형태로 충분.**

- 함수 형태: `현재 + 1`, `현재 × 2`, 토글(`on => !on`), 한 이벤트에서 여러 번 갱신
- 값 형태로 OK: `setCount(0)`(리셋)처럼 이전 값과 무관하게 특정 값 세팅

> 참고: 이 카운터의 `setCount(count + value)`도 "이전 값에 의존"한다. 지금은 클릭당 `setCount` 한 번이라 문제없지만, 더 견고한 기본형은 `setCount(c => c + value)`다.

## 한 줄 요약

한 렌더 안에서 state는 고정된 스냅샷이다. 그래서 이전 값에 기반해 갱신할 땐 `setCount(값)`이 아니라 `setCount(c => ...)`(함수형)를 써야 최신값 기준으로 누적된다.

## 실험

`setCount(count + 1)`을 한 핸들러에서 연속 두 번 호출 → 결과 관찰(`1`) → `setCount(c => c + 1)`로 바꿔 다시 관찰(`2`)하고 차이 비교.
